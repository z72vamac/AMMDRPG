Instances description

In this file, we describe the instances that we have used for the paper:

A multiple-drone arc routing and mothership coordination problem

The folder contains 10 *.json files.
The name format for each file is: AMMDRPG_numberoftargets_instancenumber.json. For instance, AMMDRPG_5_4.json is the fifth instance that has five graphs.

Each file has a Python dictionary whose keys and values are, respectively:

  • origorigorig: list of coordinates of the point defining the origin of the mothership path (or tour).
  • destdestdest: list of coordinates of the point defining the destination of the mothership path (or tour).
  • GGG: dictionary of graphs. It contains 3 dictionaries:
    • VVV: dictionary that correspond to the vertices of the graphs (represented by BegB^{e_g}Beg​ and CegC^{e_g}Ceg​ in the paper). The first value in the tuple index represents the label of the graph, the second index represents the point in the graph. The value of the dictionary is the tuple representing the coordinates of the vertex.
    • EEE: dictionary that correspond to the edges of the graphs (represented by ege_geg​ in the paper). The first value in the tuple index represents the label of the graph, the second and third, vertices labels that are joined by this edge. The value of the dictionary is the fraction of edge that must be traversed (αeg\alpha^{e_g}αeg​).
    • alphaalphaalpha: dictionary of fraction of length that must be traversed for each graph (αg\alpha^gαg).
  • vDvDvD: value of the velocity of the drone.
  • vMvMvM: value of the velocity of the mothership.
  • fleet_sizefleet\_sizefleet_size: number of available drones.
  • NNN: list of endurances considered in the paper.
